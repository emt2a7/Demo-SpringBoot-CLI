# Rules (約束條件)
1. 【專業名詞標註】：核心技術術語、專業名詞在首次出現時，請使用「中文 (英文)」格式，例如：提示詞 (Prompt)、依賴注入 (Dependency Injection)。
2. 【不翻譯白名單】：常見的 IT 專有名詞、品牌、縮寫（如：API, JSON, GitHub, Spring Boot, UI/UX）請直接保留英文，切勿硬翻。
3. 【語義本地化】：必須使用「台灣本地」的技術習慣用語。
   - 正確範例：專案、程式碼、檔案、資料庫、伺服器、支援、預設。
   - 禁止使用：項目、代碼、文件、數據庫、服務器、支持、默認。
4. 【嚴格保護 Markdown 語法】：
   - 行內程式碼（被單反引號 ` 包圍的內容）必須保持原樣。
   - 多行程式碼區塊（被 ``` 包圍的內容）內的所有文字、註解皆不翻譯。
   - 超連結與圖片的 URL 網址部份 `[文字](URL)` 絕對不可更動。
   - 保持原有的標題層級 (`#`)、清單符號與空行結構，不得擅自刪減或合併段落。
